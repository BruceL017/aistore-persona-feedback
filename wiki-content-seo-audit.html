#1
robots.txt 全站屏蔽搜索引擎爬行
Priority: 10/10 Technical SEO

问题描述

robots.txt 中设置为 User-Agent: * Disallow: /，这意味着所有搜索引擎爬虫被完全禁止访问整个网站。无论页面内容多么优质，都不会被 Google、Bing 等搜索引擎索引，organic 流量为 0。

修复建议

立即修改 robots.txt，移除 Disallow: /。至少应允许 /zh/wiki、/zh/skills、/en/wiki 等核心页面。如果站点仍在开发中，考虑只对特定测试路径使用 noindex，而非全局屏蔽 robots。

修复代码

修改 public/robots.txt：
User-Agent: *
Allow: /
Sitemap: https://aistore.club/sitemap.xml
#2
页面完全缺失 H1 和 H2 Heading
Priority: 9/10 On-Page SEO

问题描述

从 HTML 源码分析，页面没有任何 H1 或 H2 标签。所有板块标题直接从 H3 开始（如"实时快讯"、"教程精选"）。搜索引擎严重依赖 H1 来理解页面主题，缺失 H1 会大幅削弱关键词相关性和排名能力。同时，WCAG 也要求每个页面有且仅有一个 H1。

修复建议

在 Hero 区域添加一个描述性的 H1（如"AI Wiki - AI 知识库、教程与术语百科"），将现有的 H3 板块标题提升为 H2，文章/视频标题使用 H3/H4，构建正确的 H1 → H2 → H3 → H4 层级。

修复代码

1. 在页面顶部 Hero 区添加：<h1>AI Wiki - 一站式 AI 知识库、教程与术语百科</h1>
2. 将所有板块标题（实时快讯/深度好文/教程精选/术语百科/优质视频/AI 播客）从 <h3> 改为 <h2>
3. 保持文章卡片标题为 <h3>，确保层级不跳跃
#3
内容卡片使用 button 而非 a 标签，内部链接几乎为零
Priority: 8/10 SEO + Accessibility

问题描述

页面中所有文章、视频、播客卡片在静态 HTML 中都是 <button> 元素，而非 <a href="..."> 链接。整页仅有 2 个指向 wiki 的内部链接。Google 爬虫无法通过 button 发现或传递权重到子页面，所有详情页都是"孤岛页面"。

修复建议

将所有可点击的内容卡片从 <button> 改为 <a href="/zh/wiki/[slug]">，确保每个文章/视频/术语都有真实的 HTML 链接。

修复代码

将内容卡片的根元素从：
<button class="..." onClick={...}>...</button>
改为：
<a href="/zh/wiki/openclaw-intro" class="...">...</a>
#4
静态 HTML 中无子页面/详情页链接可被爬取
Priority: 8/10 Technical SEO

问题描述

页面虽有"查看全部"、"查看详情"等交互按钮，但静态 HTML 中没有任何 /zh/wiki/[slug] 形式的子页面 URL。即使 robots.txt 被修复，爬虫也无法发现这些详情页，失去了大量长尾关键词的流量入口。

修复建议

为每个内容项生成独立详情页（/zh/wiki/[slug]），并在列表页中暴露真实的链接。如果详情页尚不存在，至少应提供"查看全部"按钮指向一个可索引的列表页。

修复代码

1. 为每篇文章/术语/视频创建独立详情页 /zh/wiki/{slug}
2. 在列表页中将卡片链接到对应详情页
3. 生成 XML Sitemap 并提交到 Google Search Console

- 文章标题（<h3>）
- 1-2 行摘要
- <a href="/zh/wiki/articles/{slug}">链接</a>
#7

<button>查看全部快讯</button>

#9
视频与播客内容缺少结构化数据标记
Priority: 4/10 Schema Markup

问题描述

页面中的视频和播客只有基本的 CollectionPage Schema。缺少 VideoObject 和 PodcastEpisode Schema 意味着无法获得 Google 视频/播客富媒体结果（Rich Results），错失在搜索结果中展示缩略图、时长、发布者等信息的机会。

修复建议

为每个视频卡片添加 VideoObject JSON-LD，为每个播客卡片添加 PodcastEpisode Schema。

修复代码

为每个视频项注入 VideoObject schema：
{
  "@context": "https://schema.org",
  "@type": "VideoObject",
  "name": "Andrej Karpathy：从零手写 GPT-2",
  "description": "...",
  "thumbnailUrl": "...",
  "uploadDate": "2026-01-15"
}
#10
缺少基础安全响应头
Priority: 4/10 Technical SEO

问题描述

HTTP 响应头中缺少 Strict-Transport-Security (HSTS)、X-Content-Type-Options、X-Frame-Options 等基础安全头。缺少 HSTS 意味着浏览器不会强制升级到 HTTPS，存在降级攻击风险。Google 将网站安全性视为排名信号之一。

修复建议

在 Web 服务器或 CDN（Cloudflare）层面配置标准安全响应头。

修复代码

在 Cloudflare / Nginx / Next.js 配置中添加：
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: strict-origin-when-cross-origin

Top 改进建议

1
立即修复 robots.txt 屏蔽：这是所有 SEO 工作的前提，否则任何优化都不会被搜索引擎看到。
2
重构 Heading 层级并添加 H1：将 H3 板块标题提升为 H2，在 Hero 区补充 H1，建立清晰的信息架构。
3
将所有内容卡片改为真实链接：用 <a> 标签替换 <button>，让搜索引擎发现和传递权重到详情页。
