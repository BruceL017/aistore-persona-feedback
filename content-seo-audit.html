详细发现

#1
robots.txt 全站屏蔽搜索引擎爬虫
Priority: 10/10 Technical SEO

问题描述

当前 robots.txt 内容为 User-Agent: *\nDisallow: /，同时已生成的 sitemap.xml 也因为该规则无法被搜索引擎读取。这导致 Google、Bing 等爬虫被禁止抓取网站任何页面，一切自然搜索流量被完全切断。

修复建议

立即删除 Disallow: /，在 robots.txt 底部添加 Sitemap: https://aistore.club/sitemap.xml，仅屏蔽不需要索引的路径（如 /api/*）。

修复代码

Rewrite robots.txt to:

User-Agent: *
Allow: /
Sitemap: https://aistore.club/sitemap.xml

Optional disallows:
Disallow: /api/
Disallow: /_next/static/
#2
Canonical 标签与 Hreflang 双重指向错误
Priority: 10/10 On-Page SEO

问题描述

页面 URL 为 /zh/skills，但 canonical 指向了 https://aistore.club/zh（首页），这会让搜索引擎忽略该 skills 列表页，将所有权重合并到首页。同时 hreflang（zh → /zh、en → /en）也与当前 URL /zh/skills 不一致，进一步加剧索引混乱。

修复建议

将 canonical 改为 self-referencing：指向 /zh/skills 自身；英文版 /en/skills 同理。同时同步调整 hreflang 指向对应语言的 skills 页 URL。

修复代码

For /zh/skills:
<link rel="canonical" href="https://aistore.club/zh/skills" />
<link rel="alternate" hrefLang="zh" href="https://aistore.club/zh/skills" />
<link rel="alternate" hrefLang="en" href="https://aistore.club/en/skills" />
<link rel="alternate" hrefLang="x-default" href="https://aistore.club/zh/skills" />
#3
首屏统计数据客户端渲染导致显示 "0+"
Priority: 9/10 Content / UX

问题描述

页面源码中的 stats 初始显示为 0+ Skills 总量、0+ 作者总量、0+ 本周新增，真实数据（1000 / 599）依赖客户端 JS 请求 API 后填充。这不仅给搜索引擎爬虫留下空白/误导性内容，也严重削弱了首屏用户的信任感。

修复建议

通过 Next.js SSR / SSG / ISR 在服务端将真实统计数据直接注入 HTML 中，确保爬虫和无 JS 用户首屏即可看到 1000+ 和 599+。

修复代码

Pass real stats as initial props during SSR:

{ stats: { totalSkills: 1000, totalAuthors: 599, weeklyNew: 12 } }

Render directly in HTML:
<div>1000+ Skills 总量</div>
<div>599+ 作者总量</div>
<div>12+ 本周新增</div>
#4
零缓存策略 + TTFB 1.28s，性能基础薄弱
Priority: 8/10 Technical SEO

问题描述

响应头中设置了 Cache-Control: private, no-cache, no-store, max-age=0, must-revalidate，导致 Cloudflare 等 CDN 缓存完全无法命中，所有请求都打回源站。实测 TTFB 为 1.284s，总加载时间 1.86s，影响 Core Web Vitals 和搜索排名。

修复建议

对静态/半静态页面启用 Next.js ISR 或合理 CDN 缓存策略（如 s-maxage=60, stale-while-revalidate=86400），降低首字节响应时间。

修复代码

Enable ISR with a 60-second revalidation window:

export const revalidate = 60;

Set cache headers for semi-static content:
Cache-Control: public, s-maxage=60, stale-while-revalidate=86400

Priority: 8/10 Content Structure

URL pattern: /zh/skills/{publicId}

SSR-render the top 12 cards with real anchor links:
